この記事は、Slackで「記事書いて」と投稿しただけで生成されました。

これが、AIを活用した業務自動化の本質だ。「AIをどう活用するか」と難しく考える前に、まずは身近な繰り返し作業を任せてみる。上司へのレポート作成は、その最初の一歩として最適だ。

## なぜレポート作成から始めるのか

上司へのレポートには、いくつかの特徴がある。

- **定型的**：週次報告、進捗報告、月次サマリー。フォーマットが決まっている
- **繰り返し**：毎週、毎月、同じような作業が発生する
- **データ集約**：散らばった情報をまとめる作業が中心
- **時間を食う**：本来の業務より優先度が低いのに、意外と時間がかかる

これらの特徴は、AI自動化に向いている条件そのものだ。

```
従来のレポート作成
  ↓ 情報収集（30分）
  ↓ 整理・構造化（20分）
  ↓ 文章化（30分）
  ↓ 確認・修正（10分）
合計: 約90分

AI活用後
  ↓ Slackで一言投稿
  ↓ AIが自動生成
  ↓ 確認・微調整（10分）
合計: 約15分
```

## Claude Code Web とは

Claude Code Webは、ブラウザから直接Claudeにコーディングや自動化タスクを依頼できるサービスだ。GitHubリポジトリと連携し、コードの生成、編集、コミット、プッシュまで自動で行う。

重要なのは、これが「チャットAI」ではなく「エージェントAI」だという点だ。

**チャットAI**：質問に答える。出力はテキスト。
**エージェントAI**：タスクを実行する。出力は成果物（コード、ファイル、コミット）。

Claude Code Webは後者だ。「この記事を書いて」と言えば、実際にMarkdownファイルを作成し、GitHubにプッシュする。

## Slack連携の威力

Claude Code WebはSlackと連携できる。これにより、Slackのチャンネルから直接タスクを依頼できる。

```
Slackで投稿
  ↓
Claude Code Webがタスクを受信
  ↓
GitHubリポジトリで作業
  ↓
結果をSlackに返信
```

この記事も、まさにこの流れで作成されている。

Slackで「どうやってAIを活用するのか、とりあえず上司のレポートを自動化したらよいってテーマでclaude code web + slack アプリの題材でzennに記事をかいて」と投稿した。それだけで、この記事がZennのリポジトリにコミットされた。

## 実践：週次レポートの自動化

具体的な例を見てみよう。週次の進捗レポートを自動化する場合。

### Step 1: テンプレートを定義する

まず、レポートのフォーマットを決める。

```markdown
# 週次進捗レポート（YYYY/MM/DD週）

## 今週の成果
-

## 進行中のタスク
-

## 来週の予定
-

## 課題・相談事項
-
```

### Step 2: データソースを特定する

レポートに必要な情報がどこにあるかを整理する。

- **GitHubのPR/Issue**：今週マージしたPR、クローズしたIssue
- **Slackの投稿**：チャンネルでの報告、相談
- **プロジェクト管理ツール**：JiraやNotionのタスク状況

### Step 3: Slackから依頼する

```
@claude 今週の週次レポートを作成して。
- GitHubの main ブランチへのマージ履歴
- #dev チャンネルでの私の投稿
をもとにまとめて。
```

Claude Code Webがこれらの情報を集約し、テンプレートに沿ったレポートを生成する。

## なぜSlackが入口として優れているか

多くの企業で、Slackは業務のハブになっている。

- 常に開いている
- 情報が集約されている
- チームメンバーとのコミュニケーションの場

ここにAIへの入口を設けることで、「AIを使う」という行為が特別なものではなくなる。新しいツールを開く必要がない。いつもの場所で、いつものように投稿するだけ。

これが習慣化への近道だ。

## 「AIに仕事を奪われる」のではなく「AIに仕事を渡す」

AIの活用を躊躇する人がいる。「AIに仕事を奪われる」という不安からだ。

しかし、実態は逆だ。積極的にAIに仕事を「渡す」人が、より価値の高い仕事に集中できる。

```
AIに渡すべき仕事
- 定型的なレポート作成
- データの集約・整理
- 定期的な情報収集
- フォーマット変換

人間が集中すべき仕事
- 意思決定
- 創造的な問題解決
- 人間関係の構築
- 戦略的な思考
```

上司へのレポートを90分かけて書く代わりに、AIに任せて15分で済ませる。浮いた75分で、本当に頭を使うべき仕事に取り組む。これがAI活用の本質だ。

## 始め方

1. **Claude Code Webにアクセス**：https://claude.ai/code からアクセス
2. **GitHubと連携**：自分のリポジトリを接続
3. **Slackアプリをインストール**：ワークスペースにClaude Codeを追加
4. **まずは小さく始める**：週次レポート1つから自動化してみる

最初から完璧を目指さない。まずは1つの繰り返しタスクを自動化する。そこから徐々に範囲を広げていけばいい。
